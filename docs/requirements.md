> **バージョン**: 1.0
> **作成日**: 2026-03-13
> **ステータス**: 初版

---

## 1. はじめに

### 1.1 ドキュメントの目的

本ドキュメントは「Snow Man 出演情報リマインダーWebアプリ」の要件を定義し、設計・実装の基盤とするものである。

### 1.2 プロジェクト概要

Snow Man のファンが、メンバーのTV・ラジオ・映画出演を見逃さないためのリマインダーWebアプリを開発する。出演情報はWebスクレイピングで自動取得し、ブラウザプッシュ通知で出演前にリマインドする。

### 1.3 用語定義

| 用語 | 定義 |
|------|------|
| 出演情報 | Snow Man メンバーが出演するTV番組・ラジオ番組・映画の放送/公開情報 |
| メンバーカラー | Snow Man 各メンバーに公式に割り当てられたイメージカラー |
| Web Push | ブラウザのPush APIを利用したプッシュ通知。HTTPS環境 + Service Worker が必要 |
| PWA | Progressive Web App。ホーム画面追加・オフライン対応等を可能にするWeb技術 |
| スクレイピング | Webサイトから情報を自動的に抽出する技術 |
| VAPID | Voluntary Application Server Identification。Web Push の認証方式 |

## 2. システム概要

### 2.1 目的・背景

Snow Man は9人組のグループであり、各メンバーが個別にTV・ラジオ・映画に出演する機会が多い。全メンバーの出演情報を手動で追いかけるのは困難であるため、出演情報を自動収集し、見逃し防止のリマインダーを提供するアプリを開発する。

### 2.2 対象ユーザー

- 個人利用を想定
- ユーザー管理（ログイン・登録）は不要
- 単一ユーザーがブラウザからアクセスして利用する

## 3. 機能要件

### 3.1 出演情報の自動取得

| 項目 | 内容 |
|------|------|
| 対象メディア | TV番組、ラジオ番組、映画 |
| 取得方法 | Webスクレイピングによる自動取得（メイン: ザテレビジョン thetv.jp / 補完: 番組.Gガイド bangumi.org） |
| 取得頻度 | TV・ラジオ: 1時間ごと / 映画: 1日1回 |
| 取得情報 | 番組名、放送日時（開始・終了）、チャンネル/局名、出演メンバー、メディア種別 |
| 重複排除 | 番組名 + 放送日時 + チャンネルの複合キーで重複を判定し排除する |
| 手動取得 | 設定画面から手動でスクレイピングを実行できるボタンを提供する |

#### 3.1.1 取得対象メディア詳細

- **TV**: 地上波・BS・CSのSnow Manメンバー出演番組
- **ラジオ**: AM・FM・インターネットラジオのSnow Manメンバー出演番組
- **映画**: Snow Manメンバーが出演する映画の公開情報

#### 3.1.2 メンバー名の表記ゆれ対応

メンバー名の検索にあたり、以下の表記ゆれを考慮する:
- 漢字表記（例: 岩本照）
- ひらがな表記（例: いわもと ひかる）
- カタカナ表記（例: イワモト ヒカル）
- アルファベット表記（例: Hikaru Iwamoto）
- グループ名: Snow Man / snowman / スノーマン

### 3.2 出演情報の表示

#### 3.2.1 ダッシュボード（トップページ）

- 今日の出演情報を時系列順にカード形式で表示する
- 各カードには以下を含む:
  - 出演メンバー名（メンバーカラーのバッジ付き）
  - 番組名
  - 放送時刻（開始〜終了）
  - チャンネル/局名
  - メディア種別アイコン（TV/ラジオ/映画）
- 次の出演までのカウントダウンタイマーを表示する（「次の出演まで あと XX:XX」）

#### 3.2.2 カレンダー表示

- 月間表示・週間表示を切り替えられるカレンダーを提供する
- 出演情報をカレンダー上にイベントとして表示する
- イベントは出演メンバーのメンバーカラーで色分けする
- 複数メンバー出演の場合はグラデーションまたは複数カラーバッジで表現する

#### 3.2.3 リスト表示

- 日付ごとにグルーピングした出演情報の一覧を表示する
- 過去の出演情報も閲覧可能とする

### 3.3 フィルタリング

- **メンバー別フィルター**: 特定のメンバーの出演情報のみを表示できる
- **メディア別フィルター**: TV / ラジオ / 映画を選択して表示を絞り込める
- フィルターはカレンダー表示・リスト表示の両方で使用できる
- フィルター状態はブラウザに保存し、再訪問時に復元する

### 3.4 リマインダー通知

| 項目 | 内容 |
|------|------|
| 通知方法 | ブラウザプッシュ通知（Web Push API） |
| デフォルト通知タイミング | 出演15分前 |
| 通知内容 | メンバー名、番組名、チャンネル名、放送開始時刻 |
| 通知チェック間隔 | 1分ごとに通知対象をチェック |
| 通知済み管理 | 一度通知した出演情報には再通知しない |

### 3.5 設定機能

- **通知ON/OFF**: ブラウザプッシュ通知の有効/無効を切り替える
- **通知タイミング**: 5分前 / 15分前（デフォルト） / 30分前 から選択
- **通知対象メンバー**: メンバーごとに通知の有効/無効を設定（メンバーカラー付きチェックボックス）
- **通知対象メディア**: TV / ラジオ / 映画 ごとに通知の有効/無効を設定
- **スクレイピング状態表示**: 最終取得日時、取得件数、エラー状態を表示
- **手動スクレイピング実行**: ボタン押下で即座にスクレイピングを実行

## 4. 非機能要件

### 4.1 対応ブラウザ

| ブラウザ | バージョン |
|---------|----------|
| Google Chrome | 最新2バージョン |
| Safari | 最新2バージョン（iOS Safari含む） |
| Microsoft Edge | 最新2バージョン |
| Mozilla Firefox | 最新2バージョン |

### 4.2 レスポンシブデザイン

- スマートフォンでの利用を主な利用シーンとして想定する
- モバイルファースト設計とする
- ブレークポイント:
  - モバイル: 〜767px
  - タブレット: 768px〜1023px
  - デスクトップ: 1024px〜

### 4.3 HTTPS必須

- Web Push API の利用にはHTTPS環境が必須であるため、SSL/TLS証明書を設定する
- Let's Encrypt による無料SSL証明書を利用する

### 4.4 PWA対応

- ホーム画面への追加に対応する（Web App Manifest）
- Service Worker によるプッシュ通知受信に対応する
- オフライン時は最後に取得したデータをキャッシュから表示する

### 4.5 パフォーマンス

- ページ初回表示: 3秒以内（モバイル回線）
- カレンダー・リスト切り替え: 1秒以内
- スクレイピング処理がフロントエンドのレスポンスに影響しないこと

### 4.6 データ保持

- 出演情報は過去3ヶ月分を保持する
- それ以前のデータは自動的に削除する

## 5. 制約条件

### 5.1 スクレイピング対象サイトの利用規約遵守

- スクレイピング対象サイトの利用規約・robots.txt を確認し、許可された範囲で情報を取得する
- アクセス頻度を適切に制御し、対象サイトに過度な負荷をかけないようにする
- User-Agent を適切に設定し、botであることを明示する

### 5.2 個人利用の範囲

- 本アプリは個人利用を目的とし、商用利用は想定しない
- 取得した出演情報を第三者に再配信しない
- 著作権・肖像権に配慮し、番組内容の転載は行わない

### 5.3 技術的制約

- VPS 1台で完結する構成とする（予算: 月額500円〜1,000円程度）
- 外部APIサービス（有料）への依存を避ける
